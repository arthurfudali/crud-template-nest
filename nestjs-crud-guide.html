01

Instalação e criação do projeto

Primeiro, instale o CLI do NestJS globalmente e crie o projeto.

Terminal 
$ npm install -g @nestjs/cli
# Instala o CLI globalmente

$ nest new meu-crud
# Cria o projeto — escolha npm quando perguntado

$ cd meu-crud

$ npm run start:dev
✓  Rodando em http://localhost:3000
💡 O start:dev ativa o hot reload — o servidor reinicia automaticamente a cada alteração nos arquivos.

Agora vamos gerar um módulo para tarefas (tasks) usando o CLI:

Terminal 
$ nest generate module tasks
$ nest generate controller tasks
$ nest generate service tasks

# Atalhos equivalentes:
$ nest g mo tasks && nest g co tasks && nest g s tasks
02

Estrutura de pastas

Depois de gerar o módulo, a estrutura fica assim. Cada arquivo tem uma responsabilidade específica:

src/
  tasks/ ← nossa feature de tarefas
    tasks.module.ts ← registra tudo relacionado a tasks
    tasks.controller.ts ← recebe as requisições HTTP (rotas)
    tasks.service.ts ← contém a lógica de negócio
    task.entity.ts ← define o formato dos dados
    create-task.dto.ts ← valida o corpo das requisições
  app.module.ts ← módulo raiz da aplicação
  main.ts ← ponto de entrada, inicializa o app

📦

Module

Agrupa Controller e Service. É como uma "caixa" que organiza uma feature.

🛣️

Controller

Define as rotas (GET, POST...). Recebe a requisição e chama o Service.

⚙️

Service

Contém toda a lógica. O Controller não deve ter lógica, apenas delegação.

🗃️

Entity / DTO

Entity = formato do objeto. DTO = o que o usuário pode enviar no body.

03

Módulo — tasks.module.ts

O módulo é gerado automaticamente pelo CLI. Ele registra o controller e o service:

tasks.module.ts 
@Module({
  controllers: [TasksController],
  providers:   [TasksService],
})
export class TasksModule {}

E no app.module.ts o NestJS já adiciona automaticamente:

app.module.ts 
@Module({
  imports: [TasksModule], // ← adicionado pelo CLI
})
export class AppModule {}
04

Entidade e DTO

Crie esses dois arquivos manualmente dentro da pasta tasks/:

task.entity.ts — formato do objeto Task 
export interface Task {
  id:          string;
  title:       string;
  description: string;
  done:        boolean;
}
create-task.dto.ts — o que o usuário envia no body 
export class CreateTaskDto {
  title:       string;
  description: string;
}
💡 A entidade representa o dado completo no sistema. O DTO representa só o que o cliente pode enviar — repare que ele não tem id nem done, pois esses são gerados pelo servidor.
05

Service — tasks.service.ts

O service contém toda a lógica. Aqui usamos um array em memória (sem banco de dados):

tasks.service.ts 
import { Injectable, NotFoundException } from '@nestjs/common';
import { Task } from './task.entity';
import { CreateTaskDto } from './create-task.dto';
import { randomUUID } from 'crypto';

@Injectable()
export class TasksService {
  // "banco de dados" em memória (array simples)
  private tasks: Task[] = [];

  // CREATE — adiciona uma nova task
  create(dto: CreateTaskDto): Task {
    const task: Task = {
      id:          randomUUID(),
      title:       dto.title,
      description: dto.description,
      done:        false,
    };
    this.tasks.push(task);
    return task;
  }

  // READ ALL — retorna todas as tasks
  findAll(): Task[] {
    return this.tasks;
  }

  // READ ONE — busca por id
  findOne(id: string): Task {
    const task = this.tasks.find(t => t.id === id);
    if (!task) throw new NotFoundException(`Task ${id} não encontrada`);
    return task;
  }

  // UPDATE — atualiza title/description
  update(id: string, dto: Partial<CreateTaskDto>): Task {
    const task = this.findOne(id);
    Object.assign(task, dto);
    return task;
  }

  // DELETE — remove do array
  remove(id: string): void {
    this.findOne(id); // lança 404 se não existir
    this.tasks = this.tasks.filter(t => t.id !== id);
  }
}
06

Controller — tasks.controller.ts

O controller define as rotas e delega tudo para o service. Ele não contém lógica:

tasks.controller.ts 
import { Controller, Get, Post, Put, Delete,
         Body, Param } from '@nestjs/common';
import { TasksService } from './tasks.service';
import { CreateTaskDto } from './create-task.dto';

@Controller('tasks') // todas as rotas começam com /tasks
export class TasksController {

  // O NestJS injeta o service automaticamente (injeção de dependência)
  constructor(private readonly service: TasksService) {}

  @Post()           // POST /tasks
  create(@Body() dto: CreateTaskDto) {
    return this.service.create(dto);
  }

  @Get()            // GET /tasks
  findAll() {
    return this.service.findAll();
  }

  @Get(':id')       // GET /tasks/:id
  findOne(@Param('id') id: string) {
    return this.service.findOne(id);
  }

  @Put(':id')       // PUT /tasks/:id
  update(@Param('id') id: string, @Body() dto: CreateTaskDto) {
    return this.service.update(id, dto);
  }

  @Delete(':id')    // DELETE /tasks/:id
  remove(@Param('id') id: string) {
    return this.service.remove(id);
  }
}

Resumo dos endpoints criados:

POST /tasks Cria uma nova task
GET /tasks Lista todas as tasks
GET /tasks/:id Busca uma task por ID
PUT /tasks/:id Atualiza uma task
DELETE /tasks/:id Remove uma task
07

Testando com curl

Com o servidor rodando (npm run start:dev), teste assim:

Testando a API 
# Criar uma task (POST)
$ curl -X POST http://localhost:3000/tasks \
  -H "Content-Type: application/json" \
  -d '{"title":"Aprender NestJS","description":"Estudar o guia"}'

# Listar todas (GET)
$ curl http://localhost:3000/tasks

# Buscar por id (substitua pelo id retornado)
$ curl http://localhost:3000/tasks/SEU-ID-AQUI

# Atualizar (PUT)
$ curl -X PUT http://localhost:3000/tasks/SEU-ID-AQUI \
  -H "Content-Type: application/json" \
  -d '{"title":"NestJS dominado!","description":"Concluído"}'

# Deletar (DELETE)
$ curl -X DELETE http://localhost:3000/tasks/SEU-ID-AQUI
✅ Prefere uma interface visual? Instale o Insomnia ou Postman e faça as requisições por lá — bem mais fácil do que o curl.
